Ai-agent

一个可嵌入任意项目的 Cursor Dev Agent sidecar。支持文本 + 图片多模态对话，并展示 Agent 的思考与工具调用过程。你可以把它当成一个独立的本地服务，也可以把聊天入口嵌入到宿主项目页面里。

快速开始

1. 配置密钥 在 .env 中填入 CURSOR_API_KEY。
2. 启动服务 执行 bash run.sh，默认端口是 8765。
3. 访问首页 用当前页面确认网络、模型和 API 都正常。
4. 嵌入项目 在宿主项目模板中引入 /static/widget.js。

接口说明

浏览器能直接打开的是 GET 路由；聊天接口需要用 POST 调用。页面与 API 都支持选择模型，并可在请求体中附带图片（base64）。

  • GET /api/health：健康检查
  • POST /api/chat：同步对话（支持 images[]）
  • POST /api/chat/stream：SSE 流式对话（支持图片 + 过程事件）
  • GET /static/widget.js：可嵌入聊天组件（含图片上传）

图片上传

悬浮聊天框左下角有 📷 按钮，可选 1 张或多张图片与文本一起发送。同一会话内 Agent 会保留对话上下文，后续追问仍可参考之前上传的图片。

API 调用时在 JSON 中附带 images 数组，每项包含 data（base64）、mime_type 和可选的 name。

curl -X POST http://<ai-agent-host>:8765/api/chat \
  -H 'Content-Type: application/json' \
  -d '{
    "message": "这张截图里报错是什么意思？",
    "model": "composer-2.5",
    "images": [
      {
        "name": "screenshot.png",
        "mime_type": "image/png",
        "data": "<base64-encoded-image-data>"
      }
    ]
  }'

直接嵌入宿主项目

把下面这行插入到宿主项目页面模板的 </body> 前。宿主页面右下角会出现一个 AI 按钮。

<script src="http://<ai-agent-host>:8765/static/widget.js"></script>

如果宿主页面和 Ai-agent 不在同一域名下，可以额外传 data-api-base 指向 API 地址。

命令行测试

不嵌页面时，也可以直接用 API 验证 Agent 是否能工作。

curl -X POST http://<ai-agent-host>:8765/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"message":"先看看这个项目是干什么的","model":"composer-2.5"}'
