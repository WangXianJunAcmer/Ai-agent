Ai-agent

可嵌入任意项目的 Cursor Agent sidecar。clone 到宿主项目根目录后启动本地服务，在网页注入聊天侧边栏，Agent 直接读写宿主项目代码。

环境要求

  • Python 3.10+（cursor-sdk 硬性要求）
  • Cursor API Key（Integrations）

快速开始

# 1. 放入宿主项目
cd /path/to/your-project
git clone <this-repo-url> Ai-agent

# 2. 创建虚拟环境并安装依赖
cd Ai-agent
python3.10 -m venv ai
./ai/bin/pip install -r requirements.txt

# 3. 配置
cp .env.example .env
# 编辑 .env，填入 CURSOR_API_KEY

# 4. 启动
./ai/bin/python start.py

自检（不依赖 API Key）：

./ai/bin/python backend/check.py

确认服务正常：

curl http://127.0.0.1:8765/api/health

本页即服务首页：http://127.0.0.1:8765/

嵌入宿主项目前端

在宿主 HTML 模板的 </body> 前加一行：

<script src="http://<ai-agent-host>:8765/static/widget.js"></script>

跨域时显式传 data-api-base：

<script
  src="http://<ai-agent-host>:8765/static/widget.js"
  data-api-base="http://<ai-agent-host>:8765"
></script>

刷新宿主页后右下角出现 AI 按钮，展开侧边栏即可对话（支持图片上传）。

纯后端也可直接调 API：

curl -X POST http://<ai-agent-host>:8765/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"message":"列出项目顶层目录结构"}'

配置说明

config.yaml：

字段说明
host_project_root宿主项目根目录，相对 Ai-agent 目录
server.host / server.port本地服务地址，默认 127.0.0.1:8765
agent.modelCursor 模型，默认 composer-2.5
agent.runtimelocal（本机改代码）或 cloud
agent.cloud.repo_urlcloud 模式下的 GitHub 仓库 URL

API

  • GET /api/health — 健康检查
  • GET / — 本页（部署说明）
  • POST /api/chat — 同步对话
  • POST /api/chat/stream — SSE 流式对话（widget 使用）
  • GET /static/widget.js — 可嵌入前端组件

安全提示

  • 默认绑定便于局域网联调；公网请改回 127.0.0.1
  • Agent 可读写宿主项目并执行命令，仅用于本地开发
  • CURSOR_API_KEY 不要提交到 git
